"Build a production MCP server in Python using the MCP SDK. Expose 3 tools: postgres_query, http_api_call, file_read. Wrap it with an agent harness that sits between the Claude API and the tools. The harness must have these layers in order: (1) PermissionResolver — each tool has a risk level (read_only, network, destructive), check against a policy config before allowing; (2) ToolRegistry — validates tool name and input schema exists; (3) BudgetTracker — uses Redis to track token spend and tool call count per session, reject if over limit; (4) Executor — actually runs the tool; (5) AuditLogger — appends every call with timestamp, tool, input hash, result status, latency to PostgreSQL. The LLM never calls tools directly — it returns a structured tool_call JSON, the harness intercepts, processes through all layers, injects result back. Use FastAPI for the HTTP layer, asyncio throughout, Prometheus metrics on each layer. Include a .env config, docker-compose with postgres + redis + the server, and pytest tests for the permission and budget layers."
